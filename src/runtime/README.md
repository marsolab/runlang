# Run Runtime Library (librunrt)

The Run runtime is a C library that provides the execution environment for compiled Run programs. It is compiled into a static library (`librunrt.a`) by the Zig build system and linked into every Run binary.

## Architecture

```
Program entry (main)
  └─> run_scheduler_init()     Initialize the green thread scheduler
  └─> run_main__main()         User's main function (generated by codegen)
  └─> run_scheduler_run()      Run scheduler loop until all green threads complete
```

Generated C code calls into the runtime for memory allocation, string operations, slice management, formatted output, green thread spawning, and channel communication.

## Components

| File | Purpose |
|---|---|
| `run_alloc.c/h` | Generational memory allocation with use-after-free detection |
| `run_string.c/h` | UTF-8 string representation (pointer + length) |
| `run_slice.c/h` | Dynamic arrays with automatic growth |
| `run_fmt.c/h` | Print/format functions for primitive types |
| `run_scheduler.c/h` | Green thread scheduler with green-thread context switching |
| `run_chan.c/h` | Channel implementation with buffered and unbuffered operations |
| `run_error.h` | Error union type representation macro |
| `run_main.c` | Program entry point — calls scheduler init, user main, scheduler run |
| `run_runtime.h` | Umbrella include header for all runtime components |

## Building

The runtime is built automatically by `zig build`. The build system:

1. Compiles all `.c` files into `librunrt.a`
2. Installs headers to `zig-out/include/run/`

The compiler's C codegen backend (`codegen_c.zig`) emits `#include "run_runtime.h"` and the driver (`driver.zig`) links the generated C against `librunrt.a`.

## Sanitizer Builds

```bash
zig build -Dsanitize=true       # AddressSanitizer + UndefinedBehaviorSanitizer
zig build -Dtsan=true           # ThreadSanitizer
```

## Current Status

**Implemented:**
- Generational memory allocation (malloc-backed, 16-byte header per allocation)
- String operations (creation, concatenation, comparison, slicing)
- Slice operations (creation, append, get/set with bounds checking)
- Formatted printing for all primitive types
- Program entry point with scheduler init/run hooks
- Green-thread scheduling, yielding, and channel blocking semantics

**Current limitations:**
- Scheduler execution is currently single-P by default while multi-P queue synchronization is still being finished
- Signal-driven preemption, growable stacks, and platform support remain in progress
